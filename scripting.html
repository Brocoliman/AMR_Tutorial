Intro to Scripting

This tutorial assumes that you already know how to make a basic Marble Race. If not, go the tutorials page. When the code given has an asterisk(*) in a part, do not type the asterisk; instead follow the direction in the asterisk after the code, and it usually means that the code is custom. Note that the tutorials below are just a foundation. The contraptions you make are actually based off all of them. If you have anything to say, go here. Better, go here! This is the Intermediate level, Scripting.
To go in an object's script, go along with the following steps:
  1. Go in the Menu
  2. Find the Script Menu
  3. Don't get nervous from the size of the script; You'll soon get familiar with it

Basic Functions

Notes:

  • Some of the sections are conditions, for example, onCollide, that triggers a certain code, while some of them are variables of the object, for example, the text
  • The onCollide code is triggered when another object touches it
  • The update code is triggered every 60th of a second
  • You can create variables in the top bar below the scripting description
  • A variable lock can be used so an object, especially teleporters that teleport other teleporters, does not get any of its attributes changed. They can be made by putting a certain variable set in the update section
  • For more tips regarding the script, check out the Algodoo Scripting Wiki

Teleporter

Teleporters are the most important things in Marble Races. You may not even see it in action; it might be in the backstage!
To make a teleporter, go along with the following steps:
  1. Make 2 custom sized shapes
  2. Go in ones script menu, and in onCollide, in the "(e)=>{}"(you never put your code outside of that, unless the section doesn't have it), type: e.other.pos = my.*varx
  3. Go in the other object's script, and in the postStep, write: my.*varx = pos
  4. *The code after the asterisk is a variable, so you can customize the name, as long as both are same
  5. The teleporter will only work if an object goes in the right one, which is the one coded in Step 2. Also, the teleporter won't work on an object that doesn't have any common collision layer as the teleporter. It is best to cancel out all the collision layers of the Step 3 rectangle
  6. There!! Try it out ;)

Velocity Pads

Same as teleporters, lots of backstage action. These velocity pads apply a certain velocity(direction and speed) to an object. These are unarguably the most used element in Marble Racing.
To start, go along with the following steps:
  1. Make a custom shape(best to use a rectangle)
  2. Go in the object's script, and in onCollide, type: e.other.vel = *[x,y]
  3. *The coordinates sends the object in that x(horizontal) and y(vertical). Note that the velocity pads do not add on to the the marble's(or any colliding object's) velocity; they simply replace them.

Gravity Pads

Gravity Pads are used pretty commonly in classic races. After this section, there won't be any property changing pads. You'll have to figure it out by yourself!!!
To start, go along with the following steps:
  1. Make a custom shape(best to use a rectangle)
  2. Go in the object's script, and in onCollide, type: e.other.density = *x
  3. *The smaller the number, the higher the marble will float to! As a general guide, 0.01 keeps the marble where it is, and 0.005 makes it float up at a smooth pace
  4. Try it out!!

Variables

Variables are the building blocks of programming and scripting. They may not seem useful at the moment, but I will show you the variable magic later on...
To create a variable, go along with the following steps:
  1. Go in an object's script, and in the bar below the description, type *x=*y
  2. *This is really important!!!!! x is the name of the variable and y is the value it holds.
    • Adding "scene.my." to the left of the name will make it global
    • Making a variable global will, of course, let any object access it, and if it is change from one object, it will be changed throughout the whole scene
    • If you don't want the variable to be global, add an underscore(_) to the left of the name. This way, it only creates the variable for this object
    • If you don't add anything to the variable name, it will only appear as a variable in that property(or curly bracket)
  3. You don't have to know about all of these, but they are just for reference
    • There are 4 types of basic types of values for y: integer, float, string, boolean
    • Integers are a non-decimal numeric value. CollideSets(collision layer values) use these
    • Floats are decimal-including number. Angles might include these
    • Strings are a piece of text, put in quotation marks. Texts use these
    • Booleans are just true and false. Killer and Immortal use these
    • There are several more value types
    • Array are a list of basic value, which don't have to be the same type. Velocity and positions use them
    • Codes(the bits of scripts) can be stored as a value. OnCollide and similar properties use these
